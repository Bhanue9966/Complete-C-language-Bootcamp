# Identifiers

## 1. Introduction

Imagine a classroom with 50 students.

How does the teacher identify each student?

By using their names.

Similarly, a C program contains many elements such as:

- Variables
- Functions
- Arrays
- Structures
- Enumerations

The compiler needs a way to identify each of these elements.

These names are called **identifiers**.

An identifier is a user-defined name that uniquely identifies a program element.

---

# 2. Definition

An **identifier** is a name given by the programmer to program elements such as variables, functions, arrays, structures, unions, enumerations, and other user-defined entities.

Identifiers help both the compiler and programmers distinguish one program element from another.

---

# 3. Why Do We Need Identifiers?

Suppose a program stores:

- Student Name
- Student Age
- Student Marks

Without identifiers, the program wouldn't know which memory location belongs to which value.

Instead of saying:

```
Memory Location 1050
```

we simply write:

```c
studentAge
```

Identifiers make programs readable and easier to understand.

---

# 4. Real-Life Analogy

Think about contacts on your mobile phone.

Instead of remembering:

```

```

```

```

```

```


The name helps you recognize the contact quickly.

Similarly,

```
age

```

are identifiers that make memory locations easier to use.

# 8. Memory Representation

Example:

```c

```

Conceptually:

```
Identifier

age

↓

1000

↓

```

The identifier is simply the name used to refer to that memory location.

---

# 9. Behind the Scenes

Suppose you write:

```c
int marks = 95;
```

Conceptually:

```
Compiler Reads

↓

Keyword

int

↓

Identifier

marks

↓

Constant

95

↓

Allocate Memory

↓

Associate "marks"

↓

Store 95
```

The compiler maintains information about identifiers (such as their type and scope) so it can correctly translate the program into machine code.

# 10. Common Beginner Mistakes

### Confusing Keywords with Identifiers

❌ Incorrect

```c
int int = 10;
```

Reason:

`int` is a keyword, not an identifier.

---

### Using Meaningless Names

❌ Poor

```c
int x;
int y;
int z;
```

✔ Better

```c
int age;
int salary;
int totalMarks;
```

Meaningful identifiers improve readability.

---

### Using Similar-Looking Names

```c
int O;
int 0;
```

The letter `O` and the digit `0` can be confused.

Avoid such names whenever possible.

---

### Using Excessively Long Names

```c
int totalNumberOfStudentsWhoParticipatedInTheAnnualScienceExhibition;
```

While valid, such names reduce readability.

Choose descriptive yet concise names.

---

# 11. Language Rules

According to the C language:

- Identifiers are created by programmers.
- Identifiers are case-sensitive.
- Keywords cannot be used as identifiers.
- Every identifier must follow the naming rules (covered in the next chapter).

Examples:

```c
age
Age
AGE
```

These are three different identifiers because C is case-sensitive.

---

# 12. Best Practices

✔ Choose meaningful names.

✔ Use consistent naming throughout the program.

✔ Keep names readable.

✔ Avoid abbreviations unless they are commonly understood.

✔ Use descriptive names for functions.

Example:

❌ Poor

```c
void f();
```

✔ Better

```c
void calculateAverage();
```

# 14. Interview Questions

### 1. What is an identifier?

An identifier is a user-defined name used to identify program elements such as variables, functions, arrays, and structures.

---

### 2. What is the difference between a keyword and an identifier?

Keywords are reserved by the language, while identifiers are chosen by the programmer.

---

### 3. Can two identifiers have the same name?

Not within the same scope. Doing so results in a compilation error.

---

### 4. Are identifiers case-sensitive?

Yes.

For example:

```c
age
```

and

```c
Age
```

are different identifiers.

---

### 5. Can `printf` be used as an identifier?

Technically, C does not reserve `printf` as a keyword. However, because it is a standard library function, using `printf` as your own identifier is strongly discouraged—it can cause confusion and may lead to name conflicts.

---

# 15. Key Points to Remember

- Identifiers are programmer-defined names.
- They identify variables, functions, arrays, structures, and more.
- Keywords cannot be identifiers.
- Identifiers are case-sensitive.
- Meaningful identifiers make code easier to read and maintain.

---

# 16. Summary

Identifiers provide meaningful names for the various elements in a C program, making code easier to write, read, and maintain. Unlike keywords, which are reserved by the language, identifiers are chosen by the programmer. Selecting clear and descriptive identifiers is an essential programming skill and contributes significantly to code quality.
