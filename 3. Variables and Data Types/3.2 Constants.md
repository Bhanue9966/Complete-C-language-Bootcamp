# Constants

## 1. Introduction

In C programming, some values remain the same throughout the execution of a program.

For example:

- Number of days in a week = **7**
- Number of months in a year = **12**
- Value of Pi ≈ **3.14159**
- Maximum marks = **100**

These values should not change accidentally.

Instead of storing them in normal variables, C provides **constants**.

Using constants makes programs easier to understand and prevents accidental modification.

---

# 2. Definition

A **constant** is a fixed value that **cannot be changed during the execution of a program**.

Once a constant is defined, its value remains the same throughout the program.

---

# 3. Why Do We Need Constants?

Suppose you're writing a program to calculate the area of a circle.

Instead of writing:

```c
area = 3.14159 * radius * radius;
```

every time, you can define:

```c
const float PI = 3.14159;
```

Then use:

```c
area = PI * radius * radius;
```

Benefits:

- Easier to read
- Easier to maintain
- Avoids typing mistakes
- Prevents accidental modification

---

# 4. Real-Life Analogy

Imagine traffic signals.

```
Red   → Stop
Yellow → Wait
Green → Go
```

The meaning of each color never changes.

Similarly, constants always represent the same value throughout the program.

---

# 5. Types of Constants in C

Constants can be classified into:

```
Constants
│
├── Integer Constants
├── Floating-point Constants
├── Character Constants
├── String Constants
├── Symbolic Constants (#define)
└── const Variables
```

Let's understand each one.

---

# 6. Integer Constants

Integer constants are whole numbers without a decimal point.

Examples:

```c
10

-25

1000

0
```

Example Program

```c
#include <stdio.h>

int main()
{
    printf("%d", 100);

    return 0;
}
```

### Output

```
100
```

---

# 7. Floating-Point Constants

Floating-point constants contain decimal values.

Examples:

```c
3.14

0.5

-25.75

100.0
```

Example

```c
printf("%f", 3.14);
```

Output

```
3.140000
```

---

# 8. Character Constants

Character constants contain a single character enclosed in single quotes.

Examples:

```c
'A'

'B'

'1'

'#'
```

Example

```c
printf("%c", 'A');
```

Output

```
A
```

---

# 9. String Constants

A string constant is a sequence of characters enclosed in double quotes.

Examples:

```c
"Hello"

"C Programming"

"OpenAI"
```

Example

```c
printf("%s", "Welcome");
```

Output

```
Welcome
```

---

# 10. Symbolic Constants using `#define`

Instead of repeatedly writing the same value, you can define a symbolic constant.

### Syntax

```c
#define NAME value
```

Example

```c
#include <stdio.h>

#define PI 3.14159

int main()
{
    printf("%f", PI);

    return 0;
}
```

### Output

```
3.141590
```

### Explanation

Before compilation, the preprocessor replaces every occurrence of `PI` with `3.14159`.

Conceptually:

```c
printf("%f", PI);
```

becomes:

```c
printf("%f", 3.14159);
```

---

# 11. Constants using `const`

Another way to create constants is by using the `const` keyword.

### Syntax

```c
const data_type variable_name = value;
```

Example

```c
const float PI = 3.14159;
```

---

# 12. Example 1 – Using `const`

```c
#include <stdio.h>

int main()
{
    const int MAX_MARKS = 100;

    printf("%d", MAX_MARKS);

    return 0;
}
```

### Output

```
100
```

### Explanation

The value of `MAX_MARKS` cannot be changed after initialization.

---

# 13. Example 2 – Attempting to Modify a Constant

```c
#include <stdio.h>

int main()
{
    const int MAX = 100;

    MAX = 200;

    return 0;
}
```

### Compiler Error

```text
error: assignment of read-only variable 'MAX'
```

### Explanation

A `const` variable is read-only after it has been initialized.

### Conclusion

The compiler prevents accidental modification.

---

# 14. `#define` vs `const`

| Feature | `#define` | `const` |
|---------|-----------|---------|
| Handled by | Preprocessor | Compiler |
| Has a data type? | No | Yes |
| Occupies memory | No (text replacement) | Usually yes |
| Type checking | No | Yes |
| Preferred for typed values | No | Yes |

### Example

```c
#define PI 3.14159
```

```c
const float PI = 3.14159;
```

In modern C programming, `const` is generally preferred for typed constants because it provides type checking.

---

# 15. Memory Representation

Example:

```c
const int MAX = 100;
```

Conceptually:

```
Memory

+--------------------+
| MAX | 100 (Read Only) |
+--------------------+
```

Unlike a normal variable, the program is not allowed to modify the value of `MAX`.

---

# 16. Behind the Scenes

For a `#define`:

```c
#define PI 3.14
```

The preprocessor performs text substitution before compilation.

For a `const` variable:

```c
const float PI = 3.14;
```

The compiler treats it as a typed read-only object and prevents assignments to it.

---

# 17. Common Beginner Mistakes

### Trying to Modify a Constant

```c
const int MAX = 100;

MAX = 200;
```

This causes a compilation error.

---

### Forgetting to Initialize a `const` Variable

```c
const int MAX;
```

This is invalid because a `const` variable must be initialized when it is declared.

---

### Confusing Variables with Constants

```c
int PI = 3;
```

This creates a variable, not a constant. Its value can be changed later.

---

# 18. Best Practices

- Use `const` for values that should never change.
- Give constants meaningful names.
- By convention, many programmers write constant names in uppercase (e.g., `MAX_SIZE`, `PI`, `BUFFER_LENGTH`), although this is a style choice rather than a language rule.
- Avoid using unexplained numeric values ("magic numbers") directly in code.

Example:

❌ Bad

```c
area = 3.14159 * r * r;
```

✔ Better

```c
const float PI = 3.14159;

area = PI * r * r;
```

# 19. Interview Questions

### 1. What is a constant?

A constant is a fixed value that cannot be modified during program execution.

---

### 2. What is the difference between a variable and a constant?

A variable can change its value, while a constant cannot.

---

### 3. What is the difference between `#define` and `const`?

`#define` performs text replacement before compilation, while `const` creates a typed read-only object checked by the compiler.

---

### 4. Why should constants be used?

Constants improve readability, reduce errors, and make programs easier to maintain.

---

### 5. Can a `const` variable be modified?

No. Attempting to modify it results in a compilation error.

---

# 20. Key Points to Remember

- Constants store values that do not change.
- Integer, floating-point, character, and string literals are all constants.
- `#define` creates symbolic constants through text substitution.
- `const` creates typed read-only variables.
- Use constants instead of unexplained numeric values.

---

# 21. Summary

Constants represent fixed values that remain unchanged throughout program execution. C provides both literal constants and symbolic constants using `#define` and the `const` keyword. Choosing constants over hard-coded values improves code readability, maintainability, and reliability while helping prevent accidental modifications.
