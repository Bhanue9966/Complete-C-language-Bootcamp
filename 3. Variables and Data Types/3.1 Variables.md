# Variables

## 1. Introduction

Imagine you want to store your money.

Would you keep it in your hand forever?

No.

You keep it in a wallet, bank account, or locker.

Similarly, when a C program needs to store information such as numbers, characters, or decimal values, it requires a place in the computer's memory.

That place is called a **variable**.

Variables are one of the most important concepts in programming because almost every program uses them.

Without variables, a program cannot remember values entered by the user or results of calculations.

---

# 2. Definition

A **variable** is a named memory location used to store data that can change during the execution of a program.

The value stored in a variable may be modified whenever required.

---

# 3. Why Do We Need Variables?

Consider a calculator.

When you enter:

```
25 + 15
```

The calculator temporarily stores:

```
25

15

40
```

Somewhere in memory.

Similarly, C programs store values in variables.

Without variables:

- Input cannot be stored.
- Calculations cannot be performed.
- Results cannot be remembered.
- Programs cannot make decisions based on stored data.

---

# 4. Real-Life Analogy

Imagine a classroom with lockers.

```
Locker No. 1 → Books

Locker No. 2 → Laptop

Locker No. 3 → Bag
```

Each locker has:

- A name or number
- Storage space
- Contents that can change

A variable works the same way.

```
Variable Name

↓

Memory Location

↓

Stored Value
```

For example:

```
age

↓

Memory

↓

21
```

---

# 5. How Variables Work

Suppose you write:

```c
int age = 21;
```

The computer performs the following steps:

```
Step 1

Reserve memory

↓

Step 2

Name it "age"

↓

Step 3

Store value 21

↓

Program uses it whenever required
```

# 6. Syntax

## Variable Declaration

```c
data_type variable_name;
```

Example:

```c
int age;
```

---

## Variable Initialization

```c
data_type variable_name = value;
```

Example:

```c
int age = 21;
```

---

## Variable Assignment

```c
variable_name = value;
```

Example:

```c
age = 25;
```

---

# 7. Declaration vs Initialization vs Assignment

These three concepts are often confused by beginners.

### Declaration

Creates the variable.

```c
int age;
```

Memory is allocated, but no initial value is specified.

---

### Initialization

Assigns the first value when the variable is created.

```c
int age = 21;
```

---

### Assignment

Changes the value later.

```c
age = 25;
```

---

## Summary Table

| Operation | Example | Purpose |
|-----------|---------|---------|
| Declaration | `int age;` | Create a variable |
| Initialization | `int age = 21;` | Create and assign the first value |
| Assignment | `age = 25;` | Change the value later |

---

# 8. Example 1 – Declaring a Variable

```c
#include <stdio.h>

int main()
{
    int age;

    age = 20;

    printf("%d", age);

    return 0;
}
```

### Output

```
20
```

### Explanation

- A variable named `age` is declared.
- Memory is allocated.
- The value `20` is assigned.
- The value is displayed.

### Conclusion

Variables must be declared before they are used.

---

# 9. Example 2 – Initialization

```c
#include <stdio.h>

int main()
{
    int marks = 95;

    printf("%d", marks);

    return 0;
}
```

### Output

```
95
```

### Explanation

The variable is declared and initialized in a single statement.

---

# 10. Example 3 – Changing a Variable

```c
#include <stdio.h>

int main()
{
    int score = 50;

    printf("Initial Score = %d\n", score);

    score = 90;

    printf("Updated Score = %d", score);

    return 0;
}
```

### Output

```
Initial Score = 50
Updated Score = 90
```

### Explanation

Variables are called variables because their values can change during program execution.

---

# 11. Example 4 – Using Multiple Variables

```c
#include <stdio.h>

int main()
{
    int length = 10;
    int width = 5;

    int area = length * width;

    printf("Area = %d", area);

    return 0;
}
```

### Output

```
Area = 50
```

### Explanation

The program uses multiple variables to calculate the area of a rectangle.

Consider:

Variable Name

# 13. Behind the Scenes

When the compiler encounters:

```c
int age = 21;
```

It conceptually performs these steps:

```
Compiler

↓

Determines the data type

↓

Allocates memory

↓

Associates the name "age"

↓

Stores the value 21
```

Whenever you use `age` later in the program, the compiler generates code that accesses the corresponding memory location.

# 14. Rules for Naming Variables

A variable name:

- Can contain letters (`A-Z`, `a-z`)
- Can contain digits (`0-9`)
- Can contain underscores (`_`)
- Must begin with a letter or underscore
- Is case-sensitive

Valid examples:

```c
age

studentName

total_marks

count1

_price
```

Invalid examples:

```c
1age        // Starts with a digit

student-name // Hyphen is not allowed

total marks // Space is not allowed

int         // Keyword cannot be used
```

> We'll study identifiers, keywords, and naming rules in the next chapters.

# 15. Common Beginner Mistakes

### Using a Variable Before Declaration

❌ Incorrect

```c
age = 20;

int age;
```

✔ Correct

```c
int age;

age = 20;
```

---

### Forgetting Initialization When Needed

```c
int age;

printf("%d", age);
```

The variable contains an **indeterminate value** (often called a "garbage value" informally). Reading it before assigning a value results in **undefined behavior**.

✔ Better

```c
int age = 0;
```

---

### Redeclaring a Variable in the Same Scope

```c
int age = 20;

int age = 30;
```

This results in a compilation error because `age` is declared twice in the same scope.

---

# 16. Best Practices

- Use meaningful variable names.
- Initialize variables whenever practical.
- Keep variable names short but descriptive.
- Avoid single-letter names except for simple loop counters.
- Follow a consistent naming style throughout your program.

# 17. Interview Questions

### 1. What is a variable?

A variable is a named memory location used to store data that can change during program execution.

---

### 2. Why do we use variables?

Variables allow programs to store, update, and retrieve data while they are running.

---

### 3. What is the difference between declaration and initialization?

Declaration creates the variable, while initialization assigns its first value at the time of creation.

---

### 4. Can a variable's value change?

Yes. Unless it is declared as a constant, a variable's value can be modified during execution.

---

### 5. Is C case-sensitive?

Yes.

```c
age
```

and

```c
Age
```

are two different variables.

---

# 18. Key Points to Remember

- A variable stores data in memory.
- Every variable has a name, type, and value.
- Variables must be declared before use.
- Initialization assigns the first value.
- Assignment updates the value later.
- Choose meaningful and valid variable names.
