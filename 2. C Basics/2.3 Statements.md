# Statements

## 1. Introduction

When we write a C program, we give instructions to the computer.

For example:

- Print a message.
- Add two numbers.
- Read input from the user.
- Store a value in a variable.
- Call a function.
- Make a decision.

Each individual instruction is called a **statement**.

A C program is simply a sequence of statements that are executed one after another (unless the execution flow changes using loops or decision-making statements).

Without statements, a C program cannot perform any useful task.

---

# 2. Definition

A **statement** is a single instruction written in a C program that tells the computer to perform a specific operation.

Most statements in C end with a **semicolon (`;`)**.

---

# 3. Why Do We Need Statements?

Imagine a chef preparing a meal.

The recipe contains several instructions:

1. Wash the vegetables.
2. Cut the vegetables.
3. Heat the pan.
4. Add oil.
5. Cook the vegetables.

Each instruction is performed one after another.

Similarly, in C, each statement tells the computer to perform one specific task.

---

# 4. Real-Life Analogy

Think about following directions while traveling.

```
Start Journey

↓

Walk 100 meters

↓

Turn Left

↓

Walk 50 meters

↓

Destination
```

Each instruction helps you reach your destination.

Likewise, each statement helps the computer complete a program.

---

# 5. General Syntax

Most C statements follow this pattern:

```c
statement;
```

Example:

```c
printf("Hello");
```

The semicolon marks the end of the statement.

> **Note:** We will study semicolons in detail in the next chapter.

---

# 6. Types of Statements in C

C has several kinds of statements.

```
Statements
     │
     ├── Declaration Statements
     ├── Assignment Statements
     ├── Expression Statements
     ├── Function Call Statements
     ├── Control Statements
     ├── Jump Statements
     └── Compound Statements
```

Let's understand each one.

---

# 7. Declaration Statements

A declaration statement introduces a variable and specifies its data type.

### Syntax

```c
data_type variable_name;
```

### Example

```c
int age;
float salary;
char grade;
```

### Explanation

- `int age;` declares an integer variable.
- `float salary;` declares a floating-point variable.
- `char grade;` declares a character variable.

### Conclusion

Variables must generally be declared before they are used.

---

# 8. Assignment Statements

Assignment statements store a value in a variable.

### Syntax

```c
variable = value;
```

### Example

```c
age = 21;
salary = 35000.50;
grade = 'A';
```

### Explanation

The value on the right-hand side is assigned to the variable on the left-hand side.

### Conclusion

Assignment statements initialize or update variable values.

---

# 9. Expression Statements

An expression statement performs a calculation or evaluates an expression.

### Example

```c
sum = a + b;
```

```c
count++;
```

```c
total = price * quantity;
```

### Explanation

Expressions combine operators and operands to produce a result.

### Conclusion

Expression statements are commonly used for calculations and updates.

---

# 10. Function Call Statements

A function call statement asks a function to perform its task.

### Example

```c
printf("Hello");
```

```c
scanf("%d", &age);
```

### Explanation

The function executes and then returns control to the calling function.

### Conclusion

Functions help organize and reuse code.

---

# 11. Control Statements

Control statements change the normal flow of execution.

Examples:

```c
if

if-else

switch

for

while

do-while
```

Example:

```c
if (age >= 18)
{
    printf("Eligible");
}
```

### Explanation

The `printf()` statement executes only if the condition is true.

### Conclusion

Control statements allow programs to make decisions and repeat tasks.

---

# 12. Jump Statements

Jump statements transfer control from one part of the program to another.

Examples:

```c
break;

continue;

return;
```

Example:

```c
return 0;
```

### Explanation

`return` immediately ends the current function and optionally returns a value.

### Conclusion

Jump statements alter the normal execution sequence.

---

# 13. Compound Statements

A compound statement is a group of statements enclosed within braces.

### Example

```c
{
    int x = 10;
    printf("%d", x);
}
```

### Explanation

The braces treat multiple statements as a single block.

### Conclusion

Compound statements are used with functions, loops, and conditional statements.

---

# 14. Example 1 – Printing Output

```c
#include <stdio.h>

int main()
{
    printf("Welcome to C Programming!");

    return 0;
}
```

### Output

```
Welcome to C Programming!
```

### Statements Used

- Function call statement
- Return statement

### Conclusion

Even a simple program consists of multiple statements.

---

# 15. Example 2 – Declaration and Assignment

```c
#include <stdio.h>

int main()
{
    int age;

    age = 22;

    printf("%d", age);

    return 0;
}
```

### Output

```
22
```

### Statements Used

- Declaration statement
- Assignment statement
- Function call statement
- Return statement

### Conclusion

Different types of statements work together to complete a task.

---

# 16. Example 3 – Calculation

```c
#include <stdio.h>

int main()
{
    int a = 15;
    int b = 5;
    int sum;

    sum = a + b;

    printf("Sum = %d", sum);

    return 0;
}
```

### Output

```
Sum = 20
```

### Explanation

The assignment statement stores the result of the expression `a + b` in `sum`.

### Conclusion

Statements are executed sequentially to produce the final output.

# 18. Behind the Scenes

When the compiler reads your program, it processes one statement at a time.

For example:

```c
int x = 10;

x = x + 5;

printf("%d", x);
```

The compiler analyzes:

1. Variable declaration.
2. Variable assignment.
3. Function call.

Then it generates machine instructions that the CPU executes in the same logical order.

Although the CPU executes machine instructions rather than C statements, each C statement is translated into one or more machine instructions during compilation.

---

# 19. Common Beginner Mistakes

- Forgetting the semicolon at the end of a statement.
- Using variables before declaring them.
- Writing multiple unrelated statements on a single line, making code difficult to read.
- Confusing assignment (`=`) with equality comparison (`==`).
- Forgetting braces when grouping multiple statements under a control structure.

---

# 20. Best Practices

- Write one statement per line.
- Use meaningful variable names.
- Indent code consistently.
- Group related statements together.
- Add comments where they improve understanding.

# 21. Interview Questions

### 1. What is a statement in C?

A statement is a single instruction that tells the computer to perform a specific task.

---

### 2. Name different types of statements in C.

- Declaration statements
- Assignment statements
- Expression statements
- Function call statements
- Control statements
- Jump statements
- Compound statements

---

### 3. Why do most statements end with a semicolon?

The semicolon marks the end of a statement, allowing the compiler to identify where one statement ends and the next begins.

---

### 4. What is a compound statement?

A compound statement is a block of one or more statements enclosed within braces `{}`.

---

### 5. What is the difference between a declaration statement and an assignment statement?

A declaration statement creates a variable, while an assignment statement stores or updates a value in that variable.

---

# 22. Key Points to Remember

- A statement is a single instruction in a C program.
- Most statements end with a semicolon.
- Statements are executed sequentially unless control flow changes.
- Different types of statements perform different tasks.
- Every C program is made up of statements.

---

# 23. Summary

Statements are the fundamental building blocks of every C program. Each statement represents a single instruction, such as declaring a variable, assigning a value, performing a calculation, calling a function, or controlling the flow of execution. Understanding statements is essential because all C programs, regardless of their complexity, are composed of these basic instructions.
