# main() Function

## 1. Introduction

Every C program needs a starting point from where execution begins. Imagine reading a book—you always start from the first chapter, not from the middle or the end.

Similarly, when a C program runs, the operating system needs to know **which function should be executed first**.

In C, this starting point is the **`main()` function**.

No matter how many functions your program contains, execution always begins from `main()`.

---

# 2. Definition

The **`main()` function** is the entry point of a C program. It is the first function that is called when the program starts executing.

Every standard C program must contain **exactly one `main()` function**.

---

# 3. Why Do We Need `main()`?

Imagine organizing a sports event.

Before any match begins, there is a **main entrance gate** through which every player enters.

Similarly:

- `main()` is the **entry gate** of a C program.
- The operating system enters the program through `main()`.
- After entering `main()`, the program starts executing the statements one by one.

Without `main()`, the operating system does not know where to begin execution.

---

# 4. Real-Life Analogy

Imagine visiting a shopping mall.

No matter which shop you want to visit, you first enter through the **main entrance**.

```
Main Entrance
      │
      ▼
 Shopping Mall
      │
      ├── Clothing Store
      ├── Food Court
      ├── Electronics
      └── Cinema
```

Similarly:

```
Program Starts
       │
       ▼
    main()
       │
       ▼
 Other Functions
```

The program always enters through `main()` first.

---

# 5. Basic Syntax

The most common form is:

```c
int main()
{
    // Statements

    return 0;
}
```

Another standard form is:

```c
int main(void)
{
    return 0;
}
```

Both are valid in standard C.

- `main()` indicates no parameters (commonly accepted).
- `main(void)` explicitly indicates that the function takes no parameters.

---

# 6. Understanding Each Part

## Part 1 – `int`

```c
int
```

This is the **return type**.

It indicates that `main()` returns an integer value to the operating system.

Usually:

```c
return 0;
```

means the program completed successfully.

---

## Part 2 – `main`

```c
main
```

This is the function name.

The operating system looks specifically for a function named `main`.

Changing its name to something else, such as:

```c
int start()
```

means the program no longer has the required entry point.

---

## Part 3 – Parentheses `()`

These indicate the parameters accepted by the function.

Examples:

```c
main()
```

or

```c
main(void)
```

Later, we'll also learn:

```c
int main(int argc, char *argv[])
```

which accepts command-line arguments.

---

## Part 4 – Braces `{}`

```c
{
}
```

These define the beginning and end of the function body.

Everything inside the braces belongs to `main()`.

---

## Part 5 – Statements

Example:

```c
printf("Hello");
```

Statements are executed sequentially unless control flow changes.

---

## Part 6 – `return 0;`

```c
return 0;
```

This ends the `main()` function and returns control to the operating system.

A return value of `0` generally indicates successful execution.

---

# 7. Program Execution Flow

```
Program Starts
       │
       ▼
Operating System
       │
       ▼
Calls main()
       │
       ▼
Execute Statements
       │
       ▼
return 0;
       │
       ▼
Program Ends
```

---

# 8. Example 1 – Simple Program

```c
#include <stdio.h>

int main()
{
    printf("Welcome to C!");

    return 0;
}
```

### Output

```
Welcome to C!
```

### Explanation

1. The operating system starts the program.
2. Execution enters `main()`.
3. `printf()` displays the message.
4. `return 0;` ends the program.

### Conclusion

Execution always starts from `main()`.

---

# 9. Example 2 – Multiple Statements

```c
#include <stdio.h>

int main()
{
    printf("Line 1\n");
    printf("Line 2\n");
    printf("Line 3\n");

    return 0;
}
```

### Output

```
Line 1
Line 2
Line 3
```

### Explanation

The statements execute one after another in the order they appear.

### Conclusion

The program follows sequential execution unless instructed otherwise.

---

# 10. Example 3 – Calling Another Function

```c
#include <stdio.h>

void greet()
{
    printf("Welcome!\n");
}

int main()
{
    greet();

    printf("Learning C Programming\n");

    return 0;
}
```

### Output

```
Welcome!
Learning C Programming
```

### Explanation

Although the function `greet()` is defined before `main()`, it does not execute automatically.

It runs only because `main()` explicitly calls it.

### Conclusion

Every program starts from `main()`, not from the first function written in the file.

---

# 11. Example 4 – What Happens If `main()` Is Missing?

```c
#include <stdio.h>

void greet()
{
    printf("Hello");
}
```

### Possible Compiler/Linker Message

```
undefined reference to 'main'
```

### Explanation

The compiler may compile the source file, but during linking, the linker cannot find the program's entry point.

### Conclusion

A standard executable C program requires a `main()` function.

---

# 12. Behind the Scenes

Many beginners think execution starts directly from `main()`.

Actually, this is a simplified explanation.

The real process is:

```
Power On
      │
      ▼
Operating System
      │
      ▼
Load Executable into RAM
      │
      ▼
Startup Code (Runtime Initialization)
      │
      ▼
Call main()
      │
      ▼
Execute Program
      │
      ▼
return 0;
      │
      ▼
Cleanup
      │
      ▼
Operating System
```

Before `main()` is called, the runtime initializes the program environment, such as setting up memory and preparing the program to run.

---

# 13. Common Beginner Mistakes

- Writing `Main()` instead of `main()` (C is case-sensitive).
- Using a different function name as the entry point.
- Forgetting the braces `{}`.
- Omitting `return 0;` in educational examples.
- Thinking execution starts from the first function written in the source file.

---

# 14. Best Practices

- Always use `int main(void)` or `int main()`.
- Keep `main()` focused on coordinating the program rather than containing all the logic.
- Move reusable logic into separate functions.
- Write clean, readable code with proper indentation.

---

# 15. Interview Questions

### 1. What is the purpose of the `main()` function?

It is the entry point of a C program. Execution starts from `main()`.

---

### 2. Can a C program have more than one `main()` function?

No. A standard executable C program must have exactly one `main()` function.

---

### 3. Why does `main()` usually return an `int`?

It returns an integer status code to the operating system. A value of `0` generally indicates successful execution.

---

### 4. What happens if `main()` is missing?

The linker cannot find the program's entry point, so an executable cannot be created.

---

### 5. What is the difference between `main()` and `main(void)`?

Both are valid in modern C. `main(void)` explicitly states that the function accepts no parameters.

---

# 16. Key Points to Remember

- `main()` is the entry point of every standard C program.
- Execution always begins from `main()`.
- The operating system calls `main()` after runtime initialization.
- `return 0;` indicates successful program completion.
- There must be exactly one `main()` function in a standard executable program.

---

# 17. Summary

The `main()` function is the heart of every C program. It serves as the entry point where execution begins. The operating system calls `main()` after loading the program and performing runtime initialization. Inside `main()`, statements execute in sequence, and the function ends by returning a status code to the operating system. Understanding `main()` is essential because every C program, from simple examples to large applications, depends on it.
