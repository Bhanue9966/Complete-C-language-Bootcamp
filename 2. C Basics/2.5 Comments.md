# Comments

## 1. Introduction

When writing a program, not every piece of information is meant for the computer. Sometimes, we want to write notes, explanations, reminders, or documentation for ourselves or other programmers.

These notes are called **comments**.

Comments are ignored by the compiler, which means they do not affect how the program executes.

They are used to explain:
- What the code does
- Why a particular approach was taken
- Important assumptions
- Future improvements (TODOs)
- Documentation for functions and modules

Writing meaningful comments is considered a good programming practice because it improves the readability and maintainability of the code.

---

# 2. Definition

A **comment** is a piece of text in a C program that is ignored by the compiler and is intended only for human readers.

Comments help programmers understand the purpose and logic of the code.

---

# 3. Why Do We Need Comments?

Imagine reading a map without labels.

You may see roads, rivers, and buildings, but without labels, understanding the map becomes difficult.

Similarly, a program without comments may work correctly, but it can be difficult for others (or even yourself after a few months) to understand.

Comments make code easier to read and maintain.

---

# 4. Real-Life Analogy

Think about a textbook.

The main content teaches the subject.

The notes in the margins explain difficult concepts.

```
Textbook
───────────────
Main Content
───────────────
Margin Notes
───────────────
```

The margin notes do not change the content of the book, but they help readers understand it better.

Comments play the same role in a program.

---

# 5. Types of Comments in C

There are two types of comments:

1. Single-line comments
2. Multi-line comments

---

# 6. Single-Line Comments

Single-line comments begin with:

```c
//
```

Everything after `//` on that line is treated as a comment.

### Syntax

```c
// This is a comment
```

### Example

```c
#include <stdio.h>

int main()
{
    // Print a welcome message
    printf("Welcome to C Programming!");

    return 0;
}
```

### Output

```
Welcome to C Programming!
```

### Explanation

The comment is ignored by the compiler.

Only the `printf()` statement is executed.

### Conclusion

Use single-line comments for short explanations.

---

# 7. Multi-Line Comments

Multi-line comments begin with:

```c
/*
```

and end with:

```c
*/
```

Everything between them is ignored.

### Syntax

```c
/*
This is
a multi-line
comment.
*/
```

### Example

```c
#include <stdio.h>

int main()
{
    /*
       Display a greeting
       for the user.
    */

    printf("Hello!");

    return 0;
}
```

### Output

```
Hello!
```

### Conclusion

Use multi-line comments for longer explanations or documentation.

---

# 8. Example 1 – Explaining Code

```c
#include <stdio.h>

int main()
{
    // Declare a variable
    int age = 20;

    // Display the age
    printf("%d", age);

    return 0;
}
```

### Output

```
20
```

### Explanation

The comments describe the purpose of the statements.

The compiler ignores them completely.

### Conclusion

Comments improve code readability without affecting execution.

---

# 9. Example 2 – Program Header

```c
/*
    Program Name : Student Information
    Author       : Bhanu
    Purpose      : Display student details
*/

#include <stdio.h>

int main()
{
    printf("Student Name: Rahul");

    return 0;
}
```

### Output

```
Student Name: Rahul
```

### Explanation

Multi-line comments are commonly used at the beginning of source files to describe the program.

### Conclusion

Program headers provide useful documentation.

---

# 10. Example 3 – Temporary Code Disabling

Sometimes, programmers temporarily disable a section of code during testing.

```c
#include <stdio.h>

int main()
{
    printf("Program Started\n");

    /*
    printf("Debug Message 1\n");
    printf("Debug Message 2\n");
    */

    printf("Program Ended");

    return 0;
}
```

### Output

```
Program Started
Program Ended
```

### Explanation

The commented-out `printf()` statements are ignored by the compiler.

### Conclusion

Comments can temporarily disable code during development.

> **Note:** For version-controlled projects (e.g., Git), deleting unused code is usually better than leaving large blocks commented out.

---

# 11. Can We Nest Comments?

Consider this:

```c
/*
    This is a comment

    /*
       Another comment
    */

*/
```

This is **invalid** in C.

The compiler treats the first `*/` as the end of the outer comment.

### Conclusion

Multi-line comments cannot be nested.

---

# 12. Behind the Scenes

Before compilation, the compiler (or preprocessor, depending on the implementation) removes or ignores comments.

For example:

Source code:

```c
// Print message
printf("Hello");
```

Compiler sees:

```c
printf("Hello");
```

Comments never become part of the executable program.

---

# 13. Common Beginner Mistakes

### Writing unnecessary comments

❌ Bad

```c
x++; // Increment x
```

The code already makes this obvious.

---

### Writing outdated comments

```c
// Adds two numbers
result = a - b;
```

The comment is incorrect and misleading.

---

### Commenting every line

Too many comments make the code harder to read.

---

### Forgetting to close a multi-line comment

```c
/*
This comment never ends...
```

Compiler Error:

```text
error: unterminated comment
```

---

# 14. Best Practices

✔ Write comments that explain **why**, not just **what**.

✔ Keep comments short and meaningful.

✔ Update comments whenever the code changes.

✔ Remove obsolete comments.

✔ Use comments to explain complex logic, assumptions, or algorithms.

---

# 15. Good vs Bad Comments

## Bad Example

```c
count++; // Increase count by one
```

The code already explains this.

---

## Better Example

```c
count++; // Retry after a temporary network failure
```

This explains the reason behind the increment.

# 16. Interview Questions

### 1. What is a comment in C?

A comment is text ignored by the compiler that is written for human readers to explain or document the code.

---

### 2. How many types of comments are available in C?

Two:
- Single-line comments (`//`)
- Multi-line comments (`/* ... */`)

---

### 3. Do comments affect program execution?

No. Comments are ignored by the compiler and do not become part of the executable program.

---

### 4. Can multi-line comments be nested?

No. Standard C does not support nested multi-line comments.

---

### 5. Why are comments important?

They improve readability, documentation, maintenance, and collaboration.

---

# 17. Key Points to Remember

- Comments are ignored by the compiler.
- Use `//` for single-line comments.
- Use `/* ... */` for multi-line comments.
- Comments should explain **why**, not just **what**.
- Multi-line comments cannot be nested.
- Keep comments accurate and up to date.

---

# 18. Summary

Comments are an essential part of writing clean and maintainable C programs. Although they do not affect execution, they help programmers understand the purpose, logic, and design of the code. Good comments explain intent rather than obvious actions, making programs easier to maintain and collaborate on.
