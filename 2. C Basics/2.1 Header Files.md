# Header Files

## 1. Introduction

When writing a C program, one of the first lines you usually see is:

```c
#include <stdio.h>
```

Most beginners write this line in every program without knowing why it is required.

Questions like these are very common:

- What is `stdio.h`?
- Why do we use `#include`?
- What happens if we remove it?
- Are there other header files besides `stdio.h`?

In this chapter, you'll learn the purpose of header files, how they work, and why they are essential in C programming.

---

# 2. Definition

A **Header File** is a file that contains declarations of functions, macros, constants, and data types that can be shared among multiple C programs.

Header files usually have the **`.h`** extension.

Example:

```c
#include <stdio.h>
```

Here,

- `#include` → Preprocessor directive
- `stdio.h` → Standard Input/Output Header File

---

# 3. Why Do We Need Header Files?

Imagine you buy a television.

Along with the television, you receive a **user manual**.

The manual tells you:

- Which buttons exist
- What each button does
- How to use the television

Similarly,

A header file tells the compiler:

- Which functions are available
- How they should be called
- What arguments they accept
- What type of value they return

Without this information, the compiler cannot correctly compile your program.

---

# 4. Real-Life Analogy

Imagine a restaurant.

Before ordering food, you look at the menu.

The menu doesn't cook the food.

It simply tells you:

- What dishes are available
- Their names
- Their prices

The kitchen prepares the actual food.

Similarly,

- Header file → Menu
- Library → Kitchen
- Function → Food

The header file declares the functions, while the actual implementation exists in the library.

---

# 5. Basic Syntax

```c
#include <header_file_name>
```

Example:

```c
#include <stdio.h>
```

For user-defined header files:

```c
#include "myheader.h"
```

Notice the difference:

```
< >

↓

Standard Header Files

-------------------------

" "

↓

User-Defined Header Files
```

# 6. Commonly Used Header Files

| Header File | Purpose |
|--------------|---------|
| `stdio.h` | Standard Input and Output |
| `stdlib.h` | Memory allocation, conversions, utilities |
| `string.h` | String manipulation functions |
| `math.h` | Mathematical functions |
| `ctype.h` | Character testing functions |
| `time.h` | Date and time functions |
| `limits.h` | Integer limits |
| `float.h` | Floating-point limits |
| `stdbool.h` | Boolean type support |
| `assert.h` | Runtime assertions |

---

# 7. Understanding `stdio.h`

The most commonly used header file is:

```c
#include <stdio.h>
```

It provides declarations for functions such as:

```c
printf()

scanf()

getchar()

putchar()

fgets()

fputs()

fprintf()

fscanf()
```

Without including `stdio.h`, the compiler does not know about these functions.

---

# 8. Example 1 (Correct Usage)

```c
#include <stdio.h>

int main()
{
    printf("Hello, World!");

    return 0;
}
```

### Output

```
Hello, World!
```

### Explanation

- `stdio.h` declares the `printf()` function.
- The compiler recognizes `printf()`.
- The program compiles and runs successfully.

### Conclusion

Include the required header file before using library functions.

---

# 9. Example 2 (Missing Header File)

```c
int main()
{
    printf("Hello");

    return 0;
}
```

### Possible Compiler Output

```
warning: implicit declaration of function 'printf'
```

(Some modern compilers may treat this as an error.)

### Explanation

Since `stdio.h` is missing, the compiler does not know the declaration of `printf()`.

### Conclusion

Always include the appropriate header file before using a library function.

---

# 10. Example 3 (Using Another Header File)

```c
#include <math.h>
#include <stdio.h>

int main()
{
    double number = 25.0;

    printf("%.2f", sqrt(number));

    return 0;
}
```

### Output

```
5.00
```

### Explanation

- `math.h` declares the `sqrt()` function.
- `stdio.h` declares the `printf()` function.

### Conclusion

Different functions belong to different header files.

---

# 11. Standard vs User-Defined Header Files

## Standard Header File

Provided by the C Standard Library.

Example:

```c
#include <stdio.h>
```

The compiler searches in the system's standard include directories.

---

## User-Defined Header File

Created by the programmer.

Example:

```c
#include "student.h"
```

The compiler first searches in the current project directory.

---

# 12. How `#include` Works (Behind the Scenes)

Consider:

```c
#include <stdio.h>
```

Before compilation begins, the **preprocessor** processes this directive.

It effectively replaces the line with the contents of the header file (conceptually).

```
Your Source File
        │
        ▼
Preprocessor
        │
        ▼
Copies the required declarations
        │
        ▼
Compiler
        │
        ▼
Executable
```

> **Note:** The preprocessor includes the file's contents before the compiler starts compiling your program.

---

# 13. Difference Between Header File and Library

| Header File | Library |
|--------------|---------|
| Contains declarations | Contains implementations |
| Used during compilation | Used during linking |
| Usually `.h` files | Usually `.a`, `.lib`, `.so`, or `.dll` files |
| Tells the compiler what exists | Provides the actual executable code |

---

# 14. Common Beginner Mistakes

- Forgetting to include the required header file.
- Including unnecessary header files in every program.
- Confusing `#include` with copying source code manually.
- Using `< >` instead of `" "` for user-defined headers.
- Assuming all functions are declared in `stdio.h`.

---

# 15. Best Practices

- Include only the header files your program actually needs.
- Keep `#include` directives at the top of the file.
- Use descriptive names for your own header files.
- Avoid duplicate declarations by using header guards (covered later).

---

# 16. Interview Questions

### 1. What is a header file?

A header file contains declarations of functions, macros, constants, and data types that can be shared across multiple C source files.

---

### 2. Why do we use `#include`?

`#include` tells the preprocessor to include the contents of a header file before compilation.

---

### 3. What is the difference between `< >` and `" "` in `#include`?

- `< >` is typically used for standard library header files.
- `" "` is typically used for user-defined header files.

---

### 4. Does a header file contain function implementations?

Generally, no. It contains **declarations**. The actual implementations are usually stored in libraries.

---

### 5. What happens if `stdio.h` is not included but `printf()` is used?

The compiler may report an error or warning because it does not know the declaration of `printf()`.

---

# 17. Key Points to Remember

- Header files have the `.h` extension.
- They contain declarations, not usually implementations.
- `#include` is a preprocessor directive.
- `stdio.h` provides input/output function declarations.
- Include only the headers required by your program.

---

# 18. Summary

Header files are an essential part of C programming. They provide declarations for library functions, macros, constants, and data types that your program can use. The `#include` directive instructs the preprocessor to include these declarations before compilation. By understanding header files, you gain insight into how C programs are organized and how they interact with the C Standard Library.
