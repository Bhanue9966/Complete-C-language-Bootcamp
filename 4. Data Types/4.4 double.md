# double Data Type

## 1. Introduction

Many real-world applications require storing decimal values with **high precision**.

Examples:

- Value of Ď€ (Pi)
- Scientific measurements

A `float` can store decimal values, but it has limited precision.

When greater accuracy is required, C provides the **`double`** data type.

The word **`double`** stands for **double-precision floating-point number**.

---

# 2. Definition

A **`double`** is a basic data type in C used to store decimal (floating-point) numbers with **higher precision** than `float`.

It can represent decimal values more accurately because it typically uses more memory.

Examples:

```
3.141592653589793

15000.987654321

-25.678912345

0.000000123456
```

# 9. Format Specifier

For `printf()`:

```c
%f
```

or

```c
%lf
```

> In `printf()`, both `%f` and `%lf` print a `double` because `float` arguments are automatically promoted to `double`.

For `scanf()`:

```c
%lf
```

Example:

```c
double salary;

scanf("%lf", &salary);

printf("%lf", salary);
```

# 18. Common Beginner Mistakes

## Mistake 1

Using `%f` with `scanf()`

```c
double num;

scanf("%f", &num);
```

Incorrect.

Correct:

```c
scanf("%lf", &num);
```

---

## Mistake 2

Expecting Infinite Precision

A `double` is more precise than a `float`, but it still cannot represent every decimal value exactly.

---

## Mistake 3

Using `double` for Every Decimal Value

If your application only needs moderate precision and memory usage is important, `float` may be sufficient.

---

# 19. Best Practices

- Use `double` for high-precision calculations.
- Use `%lf` when reading a `double` with `scanf()`.
- Use `sizeof(double)` to check the size on your compiler.
- Avoid direct equality comparisons for floating-point values.
- Choose `float` or `double` based on the required precision.
