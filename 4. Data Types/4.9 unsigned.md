# unsigned Type Modifier

## 1. Introduction

In many real-world applications, values can **never be negative**.

Examples:

- Age
- Number of Students
- Number of Employees
- File Size
- Memory Size
- Number of Products
- Array Index

These values always start from **0** and increase positively.

Allowing negative numbers for such values is unnecessary.

To represent only non-negative values, C provides the **`unsigned`** type modifier.

---

# 2. Definition

`unsigned` is a **type modifier** in C that allows an integer type to store **only non-negative values (0 and positive numbers)**.

It is commonly used with:

- `char`
- `short`
- `int`
- `long`
- `long long`

Example:

```c
unsigned int count;
```

---

# 3. Why Do We Need `unsigned`?

Suppose you're storing the number of students in a college.

Possible values:

```
0

50

150

500

2500
```

Impossible values:

```
-5

-20

-100
```

Negative student counts do not make sense.

Instead of reserving memory for negative values, `unsigned` uses the entire range for positive values.

---

# 4. Real-Life Analogy

Imagine a staircase.

```
Ground Floor

↓

1st Floor

↓

2nd Floor

↓

3rd Floor
```

You can only move upward from the ground floor.

Similarly,

```
Unsigned Integer

0 → 1 → 2 → 3 → ...
```

There are **no negative values**.

# 6. Memory Size

The `unsigned` modifier **does not change the memory size**.

It only changes how the stored bits are interpreted.

Typical sizes:

| Type | Typical Size |
|------|--------------|
| unsigned char | 1 Byte |
| unsigned short | 2 Bytes |
| unsigned int | 4 Bytes |
| unsigned long | 4 or 8 Bytes |

---

# 7. Typical Value Range

Typical range for an unsigned 4-byte integer:

```
0

to

4,294,967,295
```

Notice that the entire range is used for non-negative values.

Compare this with a signed integer:

```
Signed int

-2,147,483,648

to

2,147,483,647
```

---

```

```

The bits that would normally represent negative numbers are instead used to extend the positive range.

# 9. Format Specifier

For `printf()`:

```c
%u
```

For `scanf()`:

```c

```

Example:

```c
unsigned int count;

scanf("%u", &count);

printf("%u", count);
```

# 18. Common Beginner Mistakes

## Mistake 1

Using `%d`

```c
unsigned int count = 100;

printf("%d", count);
```

Incorrect.

Correct:

```c
printf("%u", count);
```

---

## Mistake 2

Assuming Negative Values Can Be Stored

```c
unsigned int number = -10;
```

This compiles, but the stored value is **not** `-10`. It is converted according to the rules for unsigned integers, resulting in a large positive value. This often surprises beginners.

---

## Mistake 3

Using `unsigned` Everywhere

Not every integer should be unsigned.

If negative values are possible, use a signed type.

---

# 19. Best Practices

- Use `unsigned` only when negative values are impossible.
- Use `%u` with `printf()` and `scanf()`.
- Choose `signed` if negative values are meaningful.
- Be careful when mixing signed and unsigned values in expressions, as conversions can produce unexpected results.
- Use descriptive variable names.

---

# 20. When to Use `unsigned`

| Situation | Recommended |
|-----------|-------------|
| Number of Students | `unsigned int` |
| File Size | `unsigned int` (or a larger type if needed) |
| Memory Size | `unsigned int` |
| Product Quantity | `unsigned int` |
| Array Index | Often `size_t` (an unsigned type) |
| Number of Records | `unsigned int` |

---

# 21. `signed` vs `unsigned`

| Feature | `signed` | `unsigned` |
|---------|----------|------------|
| Negative Values | ✅ Yes | ❌ No |
| Positive Values | ✅ Yes | ✅ Yes |
| Zero | ✅ Yes | ✅ Yes |
| Typical 4-byte Range | -2,147,483,648 to 2,147,483,647 | 0 to 4,294,967,295 |
| Format Specifier | `%d` | `%u` |

# 22. Interview Questions

### 1. What is `unsigned`?

A type modifier that allows an integer type to store only non-negative values.

---

### 2. Does `unsigned` increase memory size?

No.

It changes only the interpretation of the stored bits.

---

### 3. Which format specifier is used for `unsigned int`?

```
%u
```

---

### 4. Why does `unsigned int` have a larger positive range?

Because no bit patterns are reserved for negative numbers.

---

### 5. Can an `unsigned int` store `-1`?

Not as a negative value. Assigning `-1` is converted to the corresponding unsigned representation, resulting in the maximum value for that unsigned type on typical systems.

---

# 23. Did You Know?

💡 The `sizeof()` operator returns a value of type `size_t`, which is an **unsigned integer type**.

That's one reason you'll often see loops like:

```c
size_t i;

for(i = 0; i < 10; i++)
{
    /* ... */
}
```

You'll learn more about `size_t` in later modules.

---

# 24. Key Points to Remember

- `unsigned` is a type modifier.
- It stores only non-negative values.
- Memory size remains the same as the corresponding signed type.
- Use `%u` for input and output.
- `unsigned` provides a larger positive range.
- Be careful when mixing signed and unsigned integers.

---

# 25. Summary

The `unsigned` modifier allows integer types to represent only non-negative values. Since all available bit patterns are used for zero and positive numbers, an unsigned type provides a larger positive range than its signed counterpart without increasing memory usage. It is commonly used for counts, sizes, indexes, and other quantities that can never be negative.
