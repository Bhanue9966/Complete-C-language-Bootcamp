## 1. Introduction

When an expression contains multiple operators, the compiler must determine **which operation to perform first**.

For example,


The compiler follows predefined rules called **Operator Precedence** and **Associativity**.

---

# 2. Definition

### Operator Precedence

**Operator precedence** determines **which operator is evaluated first** when an expression contains multiple operators.

Operators with **higher precedence** are evaluated before operators with **lower precedence**.

---

### Associativity

**Associativity** determines the **direction of evaluation** when multiple operators have the **same precedence**.

Associativity can be:

- Left to Right
- Right to Left

---

# 3. Why Do We Need Precedence and Associativity?

Suppose you're developing a billing application.

The total amount is calculated as:

```c
total = price + quantity * tax;
```

Without precedence rules, different computers might produce different results.

Precedence ensures every C compiler evaluates the expression in the same order.

# 4. Real-Life Analogy

# 11. Importance of Parentheses

Even if you know precedence rules, using parentheses improves readability.

Example:

Less readable:

```c
total = price + quantity * tax;
```

More readable:

```c
total = price + (quantity * tax);
```

Although both produce the same result, the second version makes your intention clear.

# 13. Common Beginner Mistakes

## Mistake 1

Ignoring precedence.

Incorrect expectation:

```c
10 + 5 * 2
```

Expected:

```
30
```

Actual:

```
20
```

---

## Mistake 2

Overusing parentheses unnecessarily.

```c
(((a + b)))
```

Use parentheses only when they improve readability or change evaluation order.

---

## Mistake 3

Confusing precedence with associativity.

- **Precedence** decides **which operator** is evaluated first.
- **Associativity** decides **the direction** when operators have the same precedence.

---

## Mistake 4

Assuming expressions are always evaluated left to right.

Example:

```c
2 + 3 * 4
```

Multiplication happens first because of higher precedence.

---

# 14. Best Practices

- Learn the most common precedence rules instead of memorizing the entire table.
- Use parentheses to make expressions clear.
- Do not rely on complex expressions when readability suffers.
- Break long expressions into smaller steps if needed.

# 16. Key Points to Remember

- Precedence determines **which operator is evaluated first**.
- Associativity determines **the evaluation direction** for operators of the same precedence.
- Parentheses have the highest precedence.
- Multiplication and division are evaluated before addition and subtraction.
- Assignment operators are evaluated from right to left.
- Use parentheses to make code more readable.

---

# 17. Summary

Operator precedence and associativity are fundamental concepts that determine how expressions are evaluated in C. Understanding these rules helps you write correct, predictable, and maintainable programs. While the compiler follows fixed precedence rules, using parentheses is the best practice to make your code clear and avoid misunderstandings.
