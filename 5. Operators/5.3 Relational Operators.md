# Relational Operators

## 1. Introduction

Imagine you're comparing two students' marks.

Student A = 85

Student B = 92

You may ask questions like:

- Are both marks equal?
- Is Student A greater than Student B?
- Is Student A less than Student B?

These are **comparisons**.

In C programming, comparisons are performed using **Relational Operators**.

They compare two values and return a result indicating whether the comparison is true or false.

---

# 2. Definition

A **Relational Operator** is a special symbol used to compare two operands.

The result of every relational operation is either:

```
True

or

False
```

In C,

```
True  = 1

False = 0
```

---

# 3. Why Do We Need Relational Operators?

Suppose you're developing an Online Examination System.

The program should check:

- Marks >= 35
- Age >= 18
- Password == Stored Password
- Balance > Minimum Balance

Without relational operators, these comparisons would not be possible.

---

# 4. Real-Life Analogy

Imagine a teacher checking pass marks.

```
Student Marks = 45

Passing Marks = 35

45 >= 35

↓

YES

↓

Pass
```

The comparison decides the outcome.

Similarly, C uses relational operators.

# 5. Types of Relational Operators

| Operator | Meaning | Example |
|----------|---------|---------|
| `==` | Equal to | `a == b` |
| `!=` | Not Equal to | `a != b` |
| `>` | Greater Than | `a > b` |
| `<` | Less Than | `a < b` |
| `>=` | Greater Than or Equal To | `a >= b` |
| `<=` | Less Than or Equal To | `a <= b` |

# 8. Example 1 – Equal To (`==`)

```c
#include <stdio.h>

int main()
{
    int a = 50;
    int b = 50;

    printf("%d", a == b);

    return 0;
}
```

### Output

```
1
```

### Explanation

Both values are equal.

```
50 == 50

↓

True

↓

1
```

---

### Conclusion

Use `==` when checking whether two values are equal.

---

# 9. Example 2 – Not Equal To (`!=`)

```c
#include <stdio.h>

int main()
{
    int a = 20;
    int b = 30;

    printf("%d", a != b);

    return 0;
}
```

### Output

```
1
```

### Explanation

20 is not equal to 30.

---

### Conclusion

Use `!=` to check whether two values are different.

# 16. Common Beginner Mistakes

## Mistake 1

Using assignment instead of comparison.

Incorrect

```c
if(a = b)
```

Correct

```c
if(a == b)
```

Explanation:

`=` assigns a value.

`==` compares two values.

---

## Mistake 2

Expecting output as "True" or "False"

```c
printf("%d", 5 > 3);
```

Output

```
1
```

C represents true as `1` and false as `0`.

---

## Mistake 3

Confusing `>` and `>=`

Example

```c
35 > 35
```

Output

```
0
```

Example

```c
35 >= 35
```

Output

```
1
```

---

## Mistake 4

Comparing floating-point values directly.

Example

```c
float a = 0.1 + 0.2;
float b = 0.3;

if(a == b)
```

Floating-point values may not compare exactly because of precision limitations.

---

# 17. Best Practices

- Use `==` only for comparison.
- Never confuse `=` with `==`.
- Use meaningful variable names.
- Use parentheses in complex conditions.
- Avoid direct equality checks for floating-point values when precision matters.

# 18. Interview Questions

### 1. What are relational operators?

Relational operators compare two operands and return either true (`1`) or false (`0`).

---

### 2. Name all relational operators in C.

- `==`
- `!=`
- `>`
- `<`
- `>=`
- `<=`

---

### 3. What is the difference between `=` and `==`?

`=` assigns a value to a variable.

`==` compares two values for equality.

---

### 4. What is the return value of a relational expression?

`1` if the comparison is true, otherwise `0`.

---

### 5. Where are relational operators commonly used?

- `if`
- `if-else`
- `while`
- `for`
- `do-while`

# 20. Key Points to Remember

- Relational operators compare two values.
- The result is always `1` (true) or `0` (false).
- They are widely used in decision-making statements.
- Do not confuse `=` with `==`.
- Relational operators do not modify operands; they only compare them.

---

# 21. Summary

Relational operators are used to compare two values and determine their relationship. They are essential for implementing conditions, loops, and decision-making in C programs. Every relational expression evaluates to either true (`1`) or false (`0`), making these operators fundamental to program logic.
